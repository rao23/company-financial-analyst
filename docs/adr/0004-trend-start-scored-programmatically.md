# Score Trend Start via a programmatic range check, not the LLM-judge layer

Trend Start is a threshold-tuned, discovered value (ADR-0002), so a hand-labeled eval case can't reasonably demand an exact-date match — any minor swing-detection threshold change would break passing cases for no real reason, and LLMs are unreliable at precise date/numeric comparison regardless. Each Trend eval case instead gets a hand-labeled tolerance range (`expected_trend_start_min`/`max`), and a new metric, Trend Start Accuracy, checks the agent's computed Trend Start falls inside it — a plain interval check, sitting alongside Numeric Consistency rather than the Faithfulness/Timing-awareness LLM-judge rubrics. Those LLM-judge rubrics still apply separately, extended with a criterion specific to Trend cases: whether the explanation correctly attributes the reversal cause rather than misattributing it to earlier, unrelated news that happens to fall inside the discovered window.
