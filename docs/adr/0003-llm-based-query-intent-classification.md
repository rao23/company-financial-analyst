# Classify Move vs Trend intent from wording, not a separate UI control

With both Move and Trend queries in v1 (ADR-0002), something has to decide which one a given click+question maps to. The alternative was two distinct UI entry points (e.g. a default point-click explanation plus a separate "explain the trend" control), which would need no classification at all. We're instead using a single free-text input next to the chart, with the LLM inferring intent from wording ("why did it drop here" vs "why is it trending down"), because a single natural-language box is a better user experience than forcing the user to know in advance which mode their question belongs to. The cost: LangGraph needs an explicit intent-routing node before tool selection, and the eval harness needs cases that specifically test ambiguous/misclassified wording, not just retrieval and faithfulness on already-classified queries.
