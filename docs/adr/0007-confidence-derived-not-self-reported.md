# Derive `confidence` from a deterministic rubric, not LLM self-report

The structured output schema (§10) includes a `confidence` field. The obvious implementation is to have the model report its own confidence directly — but self-reported LLM confidence is known to be poorly calibrated, typically skewing overconfident, and this field sits in the guardrails/output-validation layer where the project has otherwise consistently preferred deterministic checks (Trend Start Accuracy, Numeric Consistency, citation existence) over trusting the model's own assertions. Instead, `confidence` is computed by a rubric off signals the agent already produces during the expanding-window flow: which window tier resolved the answer (14/90/180-day), the trust_level of the primary citation (filing vs. news), and whether the price move's magnitude matches the found cause's apparent significance. A "no clear cause found" answer is not the lowest confidence bucket — it's the separate Honesty-on-no-cause case (§9) and must not be conflated with low confidence.
