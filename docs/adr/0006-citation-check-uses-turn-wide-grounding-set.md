# Citation-existence check validates against the whole turn's Grounding Set, not just the last tool call

The expanding-window retry (§8) can call retrieval tools up to three times in a single turn (14/90/180-day windows). The obvious-but-wrong implementation of the citation-existence check (§10) is to validate cited `source_id`s against only the most recent tool call's results, since that's "what the agent just retrieved." That's incorrect here: if the narrow 14-day window already surfaced the real cause, and the agent widens the window again for an unrelated reason (e.g. confirming move magnitude via `get_price_context`), a citation from that first call would be spuriously rejected as ungrounded even though it's legitimate. Instead, LangGraph state accumulates a deduped Grounding Set of every chunk ID returned by any tool call so far in the turn, and the citation check validates against that full set. The set resets at the start of the next turn.
