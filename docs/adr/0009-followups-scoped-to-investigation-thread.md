# Follow-ups are scoped to the same Investigation Date; Grounding Set spans the whole thread

The original design only specified a single click → single answer flow (§8) and didn't address follow-up questions. Follow-ups within a conversation ("was that related to the earnings report?") stay scoped to the Investigation Date of the click that started it — the agent never infers an implicit new date from wording alone. To ask about a different point in time, the user clicks again, starting a new Investigation Thread. This keeps date resolution explicit (an actual click) rather than requiring conversational temporal-reference resolution, which isn't needed for the core value prop and is a meaningfully harder problem than what's designed so far.

This sharpens what "turn" means in ADR-0006's Grounding Set: it spans the entire Investigation Thread, not each individual message. Without this, a follow-up re-citing something already established earlier in the same thread would be forced into a redundant re-retrieval just to get that chunk back into scope for the citation-existence check. The Grounding Set now resets only when a new Investigation Thread starts — i.e. the user clicks a new point — not on every message.
