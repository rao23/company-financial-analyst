# Support open-ended trend Q&A in v1, not just point-click explanations

The original design (§2/§8) only specified "click a point → explain the move," which assumes a fixed one-day scope. Real usage needs a second mode: an open-ended question like "why is it trending down?" where the relevant time window isn't given, it has to be discovered. We're building this into v1 rather than deferring it, even though it's materially harder (the Trend Start is found via backward-walking swing-point detection with a tunable noise threshold, not a deterministic fixed window), because a single-day-only explanation is a weak answer to the most natural question a user would actually ask when looking at a declining chart. This adds a second query type (Move vs Trend) that the agent/tool layer and the eval harness both need to account for as first-class, from the start, rather than retrofitted later.
