# financial_metrics stores As-Filed values only, written insert-if-absent

§6 already required this for derived EBITDA/FCF, but the same invariant applies to every XBRL-sourced figure in `financial_metrics`, not just the derived ones — and it needs an explicit ingestion rule to actually hold, not just a stated intent. SEC's bulk data sets contain the same `(cik, tag, period)` across multiple filings (a period reappears as a prior-year/prior-quarter comparative, or gets corrected via a 10-K/A). The obvious refresh implementation — upsert to the newest known value for a period — is exactly backwards here, since it would silently let a later comparative or restatement overwrite the original as-filed number. Instead: for each `(company, period, tag)`, select the value from the filing whose primary reporting period is that period, store it with its `source_accession_number`/`filed_date`, and only ever insert-if-absent on refresh, never update. Later restatements are recorded as metadata (`was_restated` + reference to the restating filing) for UI footnoting, without ever changing the charted value — the whole point of the feature is showing what the timeline looked like at the time, not the corrected-with-hindsight version.
