# Use LangGraph for agent orchestration, not PydanticAI

PydanticAI and LangGraph both sit at the same layer — agent orchestration and the tool-use loop — so this is a choice between them, not a case of needing both. PydanticAI offers a lighter, more ergonomic single-loop agent abstraction with built-in Pydantic-validated output, similar in spirit to the hand-rolled tool-use loop in §8 but with automatic validation wrapped around it. This design needs more than a single implicit loop, though: the expanding-window retry requires genuinely different routing rules for different conditions — a deterministic edge ("Grounding Set is empty → force-widen," never left to LLM discretion) alongside an LLM-judged edge ("magnitude mismatch → widen," which requires qualitative judgment) — which is a graph-shaped branching problem that maps directly onto LangGraph's explicit node/edge model, not PydanticAI's simpler default loop. The Pydantic-validated structured output PydanticAI would have provided is not lost — it's already covered directly by binding a plain Pydantic model to the LangGraph agent's final output (§10), independent of which orchestration framework wraps the loop.
